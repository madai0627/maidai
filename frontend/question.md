## 一、核心功能范围（精简版）

### 1. 后台管理（基础版）
- **题目管理**：
  - 仅支持单题添加（题干+4个选项+正确答案）
  - 支持按"分类"划分题目（如章节/模块，便于前台按分类做题）
  - 基础操作：查看所有题目、编辑题目、删除题目
  - （无需难度、解析、批量操作等）

- **数据查看**：
  - 仅展示用户总做题量、总正确率（无需详细统计图表）


### 2. 前台做题（核心流程）
- **选择分类**：用户先选一个题目分类（如"第一章"），进入该分类下的题目列表
- **做题流程**：
  - 按顺序展示该分类下所有题目（一题一页）
  - 每题展示：题干+A/B/C/D四个选项（单选）
  - 选择选项后，即时提示"对/错"，点击"下一题"继续
  - 完成所有题目后，显示本次做题结果（总题数、做对题数、正确率）
- **无计时/标记/收藏等额外功能**


### 3. 记录与错题（极简版）
- **做题记录**：仅记录用户"做过哪些题"以及"是否做对"（不记录时间、次数等）
- **错题管理**：自动收集所有做错的题目，用户可进入"错题本"重新做这些题（流程和普通做题一致）


## 二、简化后的数据模型（表结构）

只需要3张核心表，字段尽量精简：

1. **题目表（questions）**
   ```
   id: 唯一标识
   content: 题干内容（如"1+1等于几？"）
   optionA: 选项A
   optionB: 选项B
   optionC: 选项C
   optionD: 选项D
   correctAnswer: 正确答案（如"A"）
   categoryId: 所属分类ID（关联分类表）
   ```

2. **分类表（categories）**
   ```
   id: 唯一标识
   name: 分类名称（如"第一章 基础知识"）
   ```

3. **答题记录表（records）**
   ```
   id: 唯一标识
   userId: 用户ID（谁做的题）
   questionId: 题目ID（做了哪道题）
   userAnswer: 用户选择的答案（如"B"）
   isCorrect: 是否正确（true/false）
   ```
   *（注：错题本无需单独建表，通过筛选`isCorrect: false`的记录即可）*


## 三、前后端交互流程（简化版）

### 1. 前台做题流程
```
用户登录 → 选择分类 → 加载该分类下所有题目 → 逐题作答（选答案→看对错→下一题） → 完成后看结果 → 可去错题本重做错题
```

### 2. 核心接口设计（极简）
- 后台接口：
  - 题目CRUD（增删改查）
  - 分类CRUD
  - 获取用户答题统计（总题数、正确率）

- 前台接口：
  - 获取所有分类（用于选择）
  - 获取某分类下的所有题目（用于做题）
  - 提交答题结果（记录用户答案和对错）
  - 获取用户的错题列表（用于错题本）


## 四、优先级建议（新手开发）

1. **第一阶段**：实现基础功能
   - 后台：能添加题目和分类（最简单的表单即可）
   - 前台：能选择分类、逐题作答、显示对错

2. **第二阶段**：补充必要功能
   - 记录答题结果（正确/错误）
   - 实现错题本（展示所有做错的题，可重做）
   - 添加每题的分值，默认10分，答题之后展示总得分
